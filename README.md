# Observer Pattern Example

## Overview

This project is an implementation of the **Observer Design Pattern** based on concepts learned in the **Software Design and Architecture** course.

The purpose of this project is to demonstrate how the Observer pattern allows objects to communicate through a one-to-many relationship. When the state of the subject changes, all registered observers are automatically notified and updated.

## Objectives

- Apply object-oriented design principles learned in class  
- Understand and implement the Observer design pattern  
- Demonstrate loose coupling between subject and observers  
- Practice real-world software architecture concepts  

## Design Pattern Used

### Observer Pattern

The Observer Pattern defines a dependency between objects so that when one object (subject) changes state, all its dependents (observers) are notified automatically.
